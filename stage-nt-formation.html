GESTAGE Application Laravel

Stage réalisé du 24 février au 4 avril 2026 en télétravail avec NT Formation, autour de la reconception de GESTAGE, l'application de gestion des stages du BTS SIO.

Présentation du stage

Du 24 février au 4 avril 2026 — NT Formation — Stage réalisé en télétravail

🏢 Contexte

Pour mon stage de deuxième année, j'ai travaillé sur la reconception de l'application GESTAGE, un outil utilisé pour gérer les stages des étudiants du BTS SIO du Lycée Merleau Ponty.

🎯 Objectif

L'objectif était de reconstruire l'application proprement avec Laravel, en appliquant les bonnes pratiques du développement web pour améliorer la stabilité, la sécurité, la maintenabilité et l'évolution du projet.

⚠️ Problèmes identifiés

  • Bugs fréquents
  • Base de données instable
  • Code difficile à maintenir
  • Limites pour ajouter de nouvelles fonctionnalités

🔗 Projet

Le projet travaillé pendant le stage est GESTAGE, une application Laravel de gestion des stages. Le dépôt GitHub du projet est disponible dans les liens utiles de cette page.

Ce que j'ai fait

Missions réalisées pendant le stage

Installation et environnement

J'ai installé Laravel, configuré le fichier .env, connecté la base de données et lancé le serveur local.

J'ai aussi mis en place un workflow Git propre : branches par issue, commits réguliers, pull requests et revue de code.

Gérer le patrimoine informatique Organiser son développement professionnel Travailler en mode projet

Analyse du code existant et corrections

J'ai analysé la revue de code fournie par mon tuteur et étudié les incohérences entre les champs role et statut, les divergences entre migrations et modèles, ainsi que la structure du module d'authentification.

  • Mot de passe non hashé
  • Absence de validation des données
  • Risques d'exposition du mot de passe
  • Incohérences dans les formulaires
  • Absence de redirection après connexion
Gérer le patrimoine informatique Répondre aux incidents et aux demandes d'évolution

Authentification et rôles

J'ai corrigé les champs utilisés dans les formulaires, les redirections selon le rôle, le fonctionnement du logout et nettoyé le code pour supprimer les éléments inutiles.

J'ai également préparé l'intégration d'un système de rôles et permissions RBAC en étudiant Spatie HasRoles.

Répondre aux incidents et aux demandes d'évolution Gérer le patrimoine informatique Travailler en mode projet

Sécurisation de l'application

J'ai renforcé la sécurité du projet avec des validations backend, la vérification du hashage des mots de passe, la correction des risques d'exposition de données et le nettoyage des vues.

Répondre aux incidents et aux demandes d'évolution Gérer le patrimoine informatique

Intégration de l'API Sirene

J'ai intégré l'API Sirene de l'INSEE pour permettre la recherche d'entreprises via leur SIRET.

  • Tests TDD avec Http::fake()
  • Création du service SireneClient
  • Extraction et formatage des données
  • Externalisation de la clé API
  • Configuration dans config/services.php
Travailler en mode projet Répondre aux incidents et aux demandes d'évolution Gérer le patrimoine informatique

Module Entreprises

J'ai travaillé sur la logique métier liée aux entreprises et commencé l'interface de l'annuaire.

  • Création manuelle d'une entreprise avec le statut pending
  • Import automatique via SIRET avec updateOrCreate()
  • Mise à jour si l'entreprise existe déjà
  • Tests pour valider chaque scénario
  • Compteurs dynamiques, recherche simple et structure de base
Mettre à disposition un service informatique Travailler en mode projet Gérer le patrimoine informatique

Formulaire de création de stage

J'ai amélioré l'expérience utilisateur du formulaire de stage avec une logique dynamique et des contrôles sur les dates.

  • Filtrage dynamique des employés selon l'entreprise sélectionnée
  • Route JSON dédiée
  • Script JavaScript pour mettre à jour les maîtres de stage
  • Gestion des cas : aucun employé, un seul employé, plusieurs employés
  • Limitation automatique de la durée du stage à 6 semaines
  • Scripts pour empêcher les dates invalides
  • Nettoyage du StageController

Conclusion du stage

Ce stage m'a permis de travailler sur un projet réel, complexe et complet : la fiabilisation et l'amélioration de la nouvelle version Laravel de GESTAGE.

J'ai corrigé des bugs critiques, amélioré la sécurité, travaillé sur l'authentification, les rôles, l'API Sirene, le module Entreprises, le formulaire de stage et l'UX globale.

Cette expérience m'a permis de progresser en développement web, en organisation, en autonomie et en travail en mode projet.
